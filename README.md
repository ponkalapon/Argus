# Argus AI - Universal Client

Argus — это универсальный клиент для работы с AI-агентами, поддерживающий desktop (Windows/Linux/macOS) и mobile (Android/iOS через Expo) платформы.

## 🔧 Установка

### Быстрый старт (Bootstrap)
Для быстрого старта используйте **bootstrap-скрипты** из папки [`releases/bootstrap/`](releases/bootstrap/):

- **Windows:** [PC_Bootstrap.bat](releases/bootstrap/PC_Bootstrap.bat)
- **Linux/macOS:** [Linux_Bootstrap.sh](releases/bootstrap/Linux_Bootstrap.sh)
- **Android (Termux):** [Android_Bootstrap.sh](releases/bootstrap/Android_Bootstrap.sh)

Скрипты автоматически:
1. Клонируют репозиторий
2. Устанавливают зависимости
3. Обновляют приложение до последней версии

После установки запускайте приложение командой:
```bash
node argus-cli.js start  # Desktop
```

### Обновление
Для обновления до последней версии выполните:
```bash
node argus-cli.js update
```

## 📱 CLI-команды

| Команда | Описание |
|---------|----------|
| `node argus-cli.js update` | Обновить код, зависимости и пересобрать приложение |
| `node argus-cli.js start` | Запустить desktop-приложение |
| `node argus-cli.js bootstrap` | Сгенерировать bootstrap-скрипты |

## 🛠️ Сборка

## 🧠 Как работает память агента

Argus использует **двухуровневую систему памяти** для AI-агентов, аналогичную человеческой:

### 1. **Краткосрочная память (Сессионная)**
- Хранит **контекст текущего диалога** (вопросы пользователя, ответы агента, промежуточные данные).
- Очищается при завершении сессии или по тайм-ауту.
- Реализована через:
  - **`sessionStorage`** (для UI-компонентов).
  - **SQLite-базу данных** (для хранения истории сообщений в `src/services/sessionSearch.ts`).

### 2. **Долгосрочная память (Персистентная)**
- Сохраняет **ключевые факты о пользователе и окружении** между сессиями.
- Примеры:
  - Предпочтения пользователя (язык, тема интерфейса).
  - Технические детали окружения (ОС, установленные инструменты).
  - Уроки, извлеченные из прошлых ошибок (например, «не использовать `npm` для сборки на Android»).
- Реализована через:
  - **`localStorage`** (для UI-настроек).
  - **Файловую систему** (JSON-файлы в `~/.argus/memory/`).
  - **`src/services/memory.ts`** — модуль для управления памятью.

### Как это работает технически

1. **Запись в память**
   - Агент вызывает `memory.save()` при обнаружении важной информации (например, пользователь указал предпочтения).
   - Пример:
     ```typescript
     // Сохранение предпочтения языка
     memory.save({
       key: "user.language",
       value: "ru",
       type: "preference"
     });
     ```

2. **Чтение из памяти**
   - Перед ответом агент проверяет память на наличие релевантных фактов:
     ```typescript
     const language = memory.get("user.language"); // Вернет "ru"
     ```

3. **Поиск по истории**
   - Для сложных запросов (например, «Что мы делали с памятью на прошлой неделе?») используется **полнотекстовый поиск** по базе сессий:
     ```typescript
     const results = sessionSearch.query("память агента", { limit: 5 });
     ```

4. **Очистка и управление**
   - Память автоматически **архивируется** при превышении лимита (например, 10 000 записей).
   - Пользователь может вручную очистить память через настройки.

### Пример использования памяти

```typescript
// 1. Агент узнает, что пользователь предпочитает русский язык
memory.save({ key: "user.language", value: "ru", type: "preference" });

// 2. В следующей сессии агент автоматически переключается на русский
const lang = memory.get("user.language"); // "ru"
if (lang === "ru") {
  respond("Привет! Как я могу помочь?");
}

// 3. Если агент допустил ошибку, он сохраняет урок
memory.save({
  key: "lessons.npm_android",
  value: "Не использовать npm для сборки на Android — только pnpm",
  type: "lesson"
});
```

### Ограничения
- **Конфиденциальность**: Память хранится **локально** на устройстве пользователя (не отправляется в облако).
- **Объем**: Долгосрочная память ограничена 1 МБ на пользователя (настраивается).
- **Безопасность**: Чувствительные данные (токены, пароли) **никогда** не сохраняются в памяти.
